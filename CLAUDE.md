# 会話開始時の自動アクション

新しい会話が始まったら、必ず以下を最初に読む（ユーザーからの指示がなくても自動で実行する）：
1. `notes/session-log.md` を読む → 前回の続きから把握する
2. `notes/how-to.md` を読む → 操作ルールを把握する
3. 前回セッションから翌日以降経っている場合 → 「前回のログ以降に勉強しましたか？」と確認する  
   → 勉強した場合：日付・科目・内容・時間を聞いて `notes/session-log.md` と `notes/study-time.md` を更新

読み終わったら「前回の続きから始めます。〇〇が次のステップでしたね」と一言で状況を伝える。

# ブランチ（学習領域）

| フォルダ | 内容 | 優先度の性質 |
|---|---|---|
| `accounting/` | 試算表・仕訳・経理実務 | 仕事直結。アサイン次第で最優先になる |

※ 仕事のアサインが増えたらフォルダを追加していく。

# 週次プラン作成のルール（毎週日曜に実施）

## Claudeの動き方
- コーチのようにぐいぐい引き出すトーンで接すること
- ふわっとした答えには「具体的には？」と掘り下げること
- 最終的に「平日枠」と「土日枠」に分けたプランを出力すること

## 毎週ヒアリングする項目
1. **今週のアサイン** → どんなプロジェクトか、単純作業か・勉強が必要な仕事か
2. **勉強の必要性の仕分け** → アサインを踏まえて「今週勉強すべきこと」を一緒に整理
3. **使える時間の見込み** → 残業多そうか、土日は動けそうか
4. **気分・優先したい領域** → 今週どこに力を入れたいか

## プランの出力形式
```
## [日付] 週次プラン

### 今週のアサイン
- （内容と、勉強が必要かどうかの仕分け）

### 今週の優先順位
1. 
2. 
3. 

### 平日枠（通勤＋隙間時間）
- （タスクと目安時間）

### 土日枠（まとまった時間）
- （タスクと目安時間）

### 今週のゴール
- （週末にこれができていたらOK、という状態）
```

# 月次プラン作成のルール（月初に実施）

- その月の大きなテーマを1〜3個決める
- 週次プランはこの月次テーマから逆算して作る
- 月末に振り返りをして、翌月のプランに活かす
