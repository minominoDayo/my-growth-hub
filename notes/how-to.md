## 英語学習

### 勉強中に初めて見た単語を記録する
「新しい単語、〇〇」と言うだけ。  
→ `notes/vocab/new-words.md` に日付・読み・意味・出てきた場面を追記してくれる。  
→ Claudeが内容を見て **経理・簿記 / 労務・給与計算 / 英語** の3カテゴリに自動で振り分ける。  
→ `notes/vocab/` フォルダ = 分からない単語・用語を入れる箱。勉強中はここに蓄積する。  
→ `notes/vocab-trainer.html` を開けばフラッシュカード形式で復習できる（3回連続正解で「習得済み」）。

### TOEIC英単語を復習リストに記録する
「〇〇という単語追加して」と言うだけ。  
→ `english/toeic/vocabulary/review-words.txt` に日付・意味・場面を追記してくれる。

### 上級単語リストに追加する
「〇〇を上級リストに追加して」と言うだけ。  
→ `english/toeic/vocabulary/advanced-words.txt` に追記してくれる。

### 資料を読んだあとの理解度チェック
「〇〇読んだよ」と言うだけ。  
→ Claudeが確認の質問をして理解度を判定してくれる。  
→ 終わったら「メモリに入れる？」と聞いてくれる（記録する場合はYesと答える）。

### 労務・給与計算の勉強（labor-law フォルダ）
**チャプターごとに一緒に勉強する場合：**  
「給与計算ガイド、Ch1から一緒にやりたい」などと言うだけ。  
→ ClaudeがHTMLファイルの該当章を読み込んで内容を説明してくれる。  
→ 説明後に理解度チェックの質問をしてくれる。  
→ 理解できたら次の章へ進む。  
→ progress.html と メモリを自動更新してくれる。

**章ごとの着手状況を自動管理：**  
→ `progress.html` の各章タイトルに「未着手 / 着手済み / 理解済み」のバッジが表示される。  
→ 勉強を始めたチャプターは「着手済み」になり、理解度チェックOKが出たら「理解済み」に変わる。  
→ 「どこまで進んだ？」と聞けば現在の着手状況を教えてくれる。

**自分で読んだ後にチェックだけしたい場合：**  
「給与計算ガイド読んだよ」と言うだけ。  
→ Claudeが理解度チェックの質問をしてくれる。

### バックオフィス知識の勉強（back-office フォルダ）
「バックオフィスの〇〇チャプターやりたい」と言うだけ。  
→ Claudeが `back-office/` 内のPDF（ebook_経理決算.pdf・tax_all.pdf）から該当チャプターの内容を読み込む。  
→ 問題を出してくれる。  
→ 理解度が十分になったら `study-list.md` の該当項目を `[x]` に更新してくれる。  
→ チャプターは `back-office/study-list.md` の I〜XI章から選ぶ。

### 簿記3級の問題別進捗
簿記3級のスコアは自分で模擬試験を受けて手動入力する形式。  
→ `progress.html` の「簿記3級」セクションで日付・第1〜3問の点数を入力して「保存」。  
→ グラフに履歴が積み上がっていく（70点以上で緑色表示 = 合格ライン）。  
→ サンプル問題は Q1Samples・Q2Samples・Q3Samples フォルダに入っている。

### 内蔵資料を一通り終えた後の次のステップ案内
各フォルダの資料（HTML・PDF）をある程度勉強し終えたら、Claudeが自動で案内してくれる。  
→ 理解度チェックで「十分に理解できた」と判定されたタイミングで、  
　 **そのフォルダの `resources.md` に登録されているURLを提示して、さらなる練習・深掘りを促してくれる。**

**各フォルダのresources.mdの現状：**

| フォルダ | resources.mdの中身 |
|---|---|
| `bookkeeping-3/` | 仕訳100本ノックのURL（登録済み） |
| `english/toeic/` | Part別練習・模擬試験URLなど（登録済み） |
| `labor-law/` | 空。URLが見つかり次第自分で追加、またはClaudeに「〇〇のURLを labor-law の resources に追加して」と言う |
| `back-office/` | 空。同上 |

→ resources.mdが空のフォルダは「まだ登録されていません」と案内され、URLを追加するよう促される。

### 理解度チェックのルール（重要）
**知識インプット系の勉強（給与計算・労務・バックオフィス・簿記など）の理解度を測るときは、必ず「自分の言葉で説明させる」形式にする。**  
→ ○×問題・選択問題・「わかった？」の確認はしない。  
→「〇〇を自分の言葉で説明してみてください」「〇〇って何ですか？なぜそうなるか含めて」のように、アウトプットさせる。  
→ 説明できていれば理解済み。曖昧・不正確なら補足説明してもう一度。  
→ この基準でOKが出た項目だけ progress.html とメモリを更新する。

### 理解度・進捗の自動アップデート
**一緒に勉強を進める中で、Claudeが自動的に理解度と進捗をメモリに記録・更新していく。**  
→ 「どこまで進んだ？」「簿記3級の理解度は？」などと聞けば現状を教えてくれる。  
→ 試験結果や習得度が変わったタイミングでも自動更新される。  
→ 毎回ゼロから説明しなくていい。

## 今日の勉強スケジュールを決める

「今日何する？」と言うだけ。Claudeが以下の順で進めてくれる：

**ステップ①：壁打ち（ヒアリング）**  
まずClaudeが以下を聞いてくる。決まっていないことはその場で一緒に考える。  
- 今日使える時間帯・終わり時間  
- 気分や体力（疲れてる・集中できそう など）  
- やりたい領域・気になること（労務以外も全部OK）  
- 科目ごとに充てたい時間の希望（例：「労務2時間・英語30分」）  
  → 未定の場合：週次todoの残り項目を確認して、Claudeが適切な時間配分の案を出す  
- 「何でもいい」でも可。全候補を出して一緒に選ぶ。

**ステップ②：組み合わせを一緒に決める**  
1科目集中ではなく、複数科目を混ぜたスケジュールを前提に組む。  
（例：「労務90分→英語30分→簿記60分」など、飽きにくい構成にする）  
Claudeが案を出して、OKか調整するかを確認してから確定する。

**チェックリスト項目のルール（重要）**  
→ 「○○ができるようになる」のような定性的な書き方はしない。  
→ **「労務 Ch3〜8」「簿記 第1問 1セット」のように、範囲・数・章番号で書く（定量的）。**  
→ 「完了」「理解度チェック」などの文字は不要。章番号とタイトルのみ書く。  
   例: 「Ch4：勤怠締めのチェックポイント 完了」→「Ch4：勤怠締めのチェックポイント」  
→ 項目は壁打ちで決めた内容をそのまま使う。Claudeが勝手に考えない。

**ステップ②-補足：過去の勉強時間を参考にする**  
→ 「今週の勉強時間見せて」と言えばClaudeが過去の日別データを確認して提案に活かす  
→ 過去データは progress.html の「勉強時間」セクション（週間・日別グラフ）でも確認できる

**日曜日のdaily HTMLを作るときの追加ルール**  
→ ブロックの最後に「週次プラン作成（翌週）」ブロック（30分）を必ず追加する  
→ タスク：「Claudeと翌週の週次プランを作る → weekly-plans/YYYY-WXX.md に保存」

**繰り返し登場するアクティビティの標準所要時間（毎回確認不要）**  
→ スピーキング = 1回20分。複数回の場合は3回（計60分）が標準  
→ ジム = 移動含め1時間。夜（22時台）に配置  
→ ジム後は風呂30分をセットで入れる（ジム直後に配置）  
→ ネイル = 2セッションに分ける場合は「1時間＋2時間」が典型

**ステップ③：確定したらHTMLを生成**  
合意した内容で `notes/daily/YYYY-MM-DD.html` を作成する。  
→ **毎日新しい日付のファイルを作る（上書きしない）**。過去のファイルは `notes/daily/` に残り、後から見返せる。  
HTMLには以下が自動で含まれる（毎回セットで作る）：

**レイアウト：左右2ペイン構成**  
- 画面幅 900px 以上 → 左右に分割  
- 画面幅 900px 以下 → 上下に切り替わる（スマホ・小画面対応）
- 左ペインの内容は `.left-inner` でラップする（max-width なし・`.pane-wrap` に `padding: 0 20px` で外余白を確保する）

**左ペイン：今日の勉強タスク**  
- **現在時刻**（大きくリアルタイム表示）  
- **ブロックカウントダウン**（各ブロックの「▶ 開始」ボタンで起動、一時停止・リセット可）  
- **累計勉強時間カウンター**（休憩ブロック開始中は自動停止。タブを閉じても記録が消えない）  
- **アラーム音**（ブロック終了時にビープ音で通知）  
- **チェックリスト**（勉強ブロックのみ。休憩ブロックにはチェックリストを入れない）  
- **教材リンク**：簿記ブロックには該当PDFへのリンク、労務ブロックには給与計算ガイドへのリンクを貼る  
  → チェックリスト項目はあくまで参考。実際の勉強報告をもとに随時内容・時間配分を修正する

**プライベートTODOの扱い**
- `progress.html` の「プライベートTODO」セクションで短期タスクを管理する（localStorage保存）。
- daily HTML を作るときは、右ペインに「プライベートTODO」セクションも追加する。
  - その日やりたいこと（当日入力、日付キーで保存）
  - 短期消化タスク（progress.html と同じキーを参照して連動）
  - 新しいタスクを追加できる入力フォーム

## 勉強時間の管理（progress.html）

- `progress.html` の「月間勉強時間」セクションで科目ごとの累計時間を管理する。
- 科目分類：労務・給与計算 / 簿記3級 / 英語・TOEIC / バックオフィス / スピーキング
- **勉強時間の入力方法**：「〇〇時間追加して（労務）」と言うだけ、または progress.html 上で直接手入力。
- localStorage キー形式：`studytime_YYYY_MM_subject`（例：`studytime_2026_05_labor`）
- daily.html の科目別タイマーが終わったら、同じキーに累計で加算する。

### データ保存のしくみ（重要）

`notes/study-time.md` が「正」のデータ。ブラウザのストレージはあくまで補助。

- **ブラウザ側（localStorage）**：ページを開いている間の一時保存
- **ファイル側（study-time.md）**：永続保存。Claudeが「勉強終わり」時に更新
- **progress.html の BASELINE**：study-time.md の累計値をここに反映しておく（ストレージが消えても正しい値が表示される）

### daily HTML 生成のルール

**daily HTML を作る前に必ず以下を読む：**
1. `notes/session-log.md` → 完了済みチャプターを確認
2. `notes/study-time.md` → 現在の累計勉強時間を確認

確認した内容を HTML に「初回起動時に自動セット」するコードとして埋め込む：
- 完了済みチャプターは localStorage キーとして初期化
- 既にやったPDFは初期チェック済み状態にする
→ これで「ウィンドウを閉じてもリセットされない」状態を初回から保証できる

## 勉強を終わるとき

「勉強終わり」と言うだけ。Claudeが自動で以下を実施する：

1. **その日のdaily HTML のチェック状態を確認**  
   → 完了しているタスクを把握し、progress.html の対応キー（`kyuyo_ch*`、`roumu_ch*`、`pdf_*` など）を更新  
   → 未完了のタスクは次の「次にやること」に引き継ぐ

2. **pendingVocab の整理**  
   → daily HTML で追加した単語がある場合：ユーザーにリストを確認してもらい、カテゴリ（経理・労務・英語）に分類して `notes/vocab/new-words.md` に追記

3. **session-log.md を上書き更新**（追記ではなく全体を書き直す）  
   → 過去の内容は消して、以下3セクションで最新状態に上書き：  
   　「やったこと」「次にやること」「現在の進捗状態」  

4. **「次のセッションでは〇〇から始めましょう」と一言まとめて終了**

## 作業前の確認ルール（重要）

**ファイルの生成・編集・作成を行う前に、必ず内容をユーザーに提示して確認を取る。**
→ 変更内容・ファイル名・構成を箇条書きで見せて「これでいいですか？」と聞く。
→ OKをもらってから実行する。勝手に進めない。

**複数の変更を実施するときは、処理負担の少ない順（小→大）に実行する。**
→ テキストファイル編集 → HTML小変更 → HTML大変更・機能追加の順。
→ 実行前に変更一覧をリスト化してユーザーに確認を取ってから進める。

## このファイルの自動更新ルール（重要）

**会話の中で新しい好みや仕様が出てきたら、Claudeが自動でこのファイルを更新する。**  
明示的に「how-toに入れて」と言わなくても、以下のようなケースは自動追記・更新対象：

- 出力物（HTMLなど）に「〇〇があるといい」「〇〇にして」と仕様追加があったとき  
- 作業の進め方・順番について新しいルールや好みが出てきたとき  
- 「毎回〇〇して」「いつも〇〇にして」という習慣化の指示があったとき  

更新したときは必ず「how-to.md を更新しました：〇〇を追記しました」と一言報告する。

### 学習コンテンツを作成・更新したときの自動アクション（重要）

**新しい学習ページを作ったとき・既存ページを更新したときは、必ず以下をセットで行う：**
- `progress.html` の対応セクションに「開く」リンクボタン（`.sec-link`）を追加または更新する
- `notes/how-to.md` のファイル一覧テーブルに該当ファイルを追記する

→ ユーザーから「リンク貼って」「一覧に追加して」と言わせない。コンテンツ作成と同じタイミングで自動的に実施する。
